Chrome Dino Game

ShanghaiTech University, Computer Architecture I (CS110)

Chrome Dino pixel game on Longan Nano development board.

In this project, we implemented a Chrome Dino Runner Game using the Longan Nano development board. We utilized PlatformIO to interact with the development board and support cross-language programming. For example, we implemented most of the framework by C, but some of the auxiliary functions are implemented by RISC-V assembly. The functions can be invoked bidirectionally.

The game we implemented features:

  • A simple startup menu and diffuculty selection page.
  • A score board.
  • Randomly generated cactus and pterosaurs acting like obstacles.
  • The Dino can jump or creep controlled by the 2 buttons.
  • Super mode in which the dino can hit everything without failing the game.
View source code on GitHub: github.com/billhu0/CS110/tree/main/Projects/Project4

Multi-armed Bandit Analysis

This is the course project of SI140 (Probability and Statistics), Fall 2021.

This project is about Multi-armed Bandit, a classical problem in reinforcement learning. It contains three main parts:

  • Simulation. We have to simulate the multi-armed bandit via different algorithms under the set of given parameters and show the output, i.e., the performance of the bandit.
  • Analysis. We have to analyze the impact of each hyperparameter of each algorithm. Additionally, we have to analyze an important problem in the multi-armed bandit program, which is named the exploration-exploitation trade-off.
  • Design. We have to handle some variants of the classical scenario, including dependency and constraint, and design some algorithms by ourselves to address the novel problem variants.

You can view the project and project report on GitHub: github.com/billhu0/SI140-Project

Ray Tracing NURBS Surfaces

ShanghaiTech University, Computer Graphics I (CS171)

As a non-analytical surface, high-quality-and-efficiency NURBS surface rendering is a very challenging topic in Computer Graphics. Traditional triangulation (tessellation) method requires a huge amount of calculation and memory to generate and store meshes respectively. To solve this problem, in this project, we implemented a path tracer with global illumination to directly render untrimmed NURBS surfaces without tessellating the surface.

Following the methods proposed by Martin et al, we successfully implemented the renderer with more than 3300 C++ baseline. In details, we applied surface refinement for better rendering quality and efficiency, and constructed a BVH to maintain the surface sub-patches and accelerate the ray-patch intersection process.
